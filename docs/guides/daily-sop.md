# 日常操作 SOP

## 每日工作流程

| 时间 | 阶段 | 操作内容 |
|------|------|----------|
| 09:00 | 晨间检查 | 1. 检查各节点 SSH 连接<br>2. git pull 同步最新代码<br>3. 查看 tasks/in-progress/ 进度 |
| 09:15 | 站会 | 1. 回顾昨日完成的 Task<br>2. 确认今日优先级<br>3. 解决阻塞问题 |
| 09:30 | 任务分发 | 1. 创建今日 Task 文件<br>2. SSH 下发到各节点<br>3. 将 Task 从 todo 移到 in-progress |
| 10:00 | 架构工作 | 1. 审查架构文档<br>2. 处理技术选型问题<br>3. 更新 API 规范 |
| 14:00 | 进度检查 | 1. 检查各节点 git push 状态<br>2. 协助解决技术阻塞<br>3. 调整任务优先级 |
| 17:00 | 代码审查 | 1. Review 所有待合并的 PR<br>2. 合并通过审查的代码<br>3. 标记审查意见 |
| 17:45 | 日报总结 | 1. 移动完成的 Task 到 done/<br>2. 更新项目进度文档<br>3. 规划明日任务 |
