# AI-CRM 系统架构设计

## 1. 整体架构
采用前后端分离 + 微服务思想，后端 Python FastAPI，前端 React。

```
┌──────────────────────────────────────────────────────────┐
│                     Nginx (反向代理)                        │
├────────────────────┬─────────────────────────────────────┤
│  React SPA         │  FastAPI REST API                    │
│  Port: 3000        │  Port: 8000                          │
│  PC2 @ .169 (WIN11)│  PC3 @ .170 (macOS)                  │
├────────────────────┴─────────────────────────────────────┤
│              MySQL 8.0 + Redis 7                           │
│              (PC3 管理)                                    │
├──────────────────────────────────────────────────────────┤
│           DeepSeek API (AI 引擎)                           │
│           (PC1 管理 Token)                                 │
└──────────────────────────────────────────────────────────┘

v3.1.0 审查链 (三闸门):
  PC3 代码提交 ──→ PC4 安全审查 (.171) ──→ PC5 QA 测试 (.253) ──→ PC1 终审 (.168)
                   第一闸门                第二闸门                第三闸门
                   OWASP Top 10           功能+性能+回归           Go/No-Go
```

## 2. 技术选型
| 层面 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| 后端框架 | FastAPI | 0.115+ | 异步高性能，自动 Swagger 文档 |
| ORM | SQLAlchemy | 2.0+ | Python 最成熟的 ORM |
| 数据库 | MySQL | 8.0 | 企业级，事务支持 |
| 缓存 | Redis | 7.x | 高性能，支持多种数据结构 |
| 前端框架 | React | 18.x | 生态最强，组件化 |
| UI 库 | MUI | 5.x | 企业级组件库 |
| 图表 | ECharts | 5.x | 功能最全的图表库 |
| AI SDK | LangChain + openai | latest | DeepSeek API 兼容 OpenAI 格式 |
| 认证 | JWT + OAuth2 | - | 无状态，适合分布式 |
